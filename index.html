Product Vision
FlowForge is an AI-powered development lifecycle tool that takes a business requirement from discovery through specification, code generation, and automated code review — all within Claude Code. It replaces the manual steps at every phase: upstream BA research, spec authoring, plan review, code generation, and the solo pre-analysis step before peer review — so engineers spend their time on judgment, not mechanics. Three integrated phases cover the full journey: BA & Requirements Pipeline (CIL-6148) closes the upstream 6–10h gap, Code Generation (PROD-13009) drives the build pipeline with 15+ agents, and Code Review (PROD-13861) automates the gated IIE pre-analysis so the first human touch is always peer review.

PROD-13861 — Replace Human Code Analysis Step with Automated IIE Pipeline

Today's pipeline ends its automated phase at /ff.verify. After that, a solo engineer reads the diff, assesses correctness, spots architectural issues, and writes up findings before peer review starts. This preparatory analysis step is largely mechanical — spec alignment, Go anti-patterns, security/GDPR surface, testability — and takes significant time per feature. /flowforge.review is extended in-place into a gated, evidence-driven replacement for this solo human pre-analysis step, so that the first human touch in the review cycle is peer code review, not solo pre-analysis by one engineer. The 2-approver peer review is not eliminated — only the preparatory solo analysis phase is automated.

IIE Isolation
The Independent Implementation Examiner runs in a named non-fork subagent with an isolated context window. It cannot see builder conclusions, cannot run tools, and is structurally prevented from self-assessment contamination.
Circuit Breaker
Maximum 2 automated fix-and-re-review cycles. On cycle 2 with remaining critical/high findings the orchestrator emits ESCALATE and REQUEST_MANUAL_FIX — the loop does not continue.
Gate Conditions
PASS verdict requires all three simultaneously: (a) zero critical findings, (b) zero undismissed high findings, (c) 100% AC traceability. Passing tests alone do not satisfy these conditions.
Policy Routing
Each finding is mapped to exactly one action by severity × confidence. Architecture, GDPR, and scope findings always escalate regardless of severity. Medium findings that auto-fix are flagged requires_human_confirmation for peer reviewers.

The Independent Implementation Examiner is structurally isolated — it cannot see builder conclusions, cannot run tools, and is adversarially oriented to disprove readiness.
