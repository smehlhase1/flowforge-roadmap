What FlowForge can do NOW

Dev Pipeline — Fully Operational

FlowForge v1.1.1 covers the full developer loop from spec capture to a merged PR with automated CI feedback.

  • Interactive spec capture with Gap Reviewer (mid-capture), Self-Review, and QA Author agents
  • Jira ticket creation, quality gate validation, and branch naming enforcement at /ff.require
  • 6-agent plan review (5 specialists + Devil's Advocate) via /ff.plan
  • Code generation across Standard, Express, and Fast lanes via /ff.build
  • 4-verifier + Devil's Advocate compliance check via /ff.verify
  • Auto-fix loop and self-healing via /ff.fix
  • GitHub Actions integration: auto-trigger from Jira status change, post-back on CI failure
  • Automated Java QA test generation alongside Go code
  • Claude usage cost tracking pushed to Prometheus
  • Stale-context guard: prompts user to pull latest main branch before generation
  • Related Jira tickets and branches passed as context into code generation
  • Ticket quality gate blocks low-quality tickets before generation starts
  • Git branch rebasing for FlowForge-generated branches
  • Enforced git branch naming template

Developer Velocity

Reduce time from spec to merged PR
Lane-based generation (Standard / Express / Fast)
Teams choose build depth per ticket — routine tasks use Fast lane to skip full review cycles, cutting multi-hour manual effort to minutes.
Available Now
Stale-context guard & automatic branch rebase
Prevents wasted generation runs on outdated main — prompts pull before build, and rebases existing FlowForge branches to latest main automatically.
Available Now
Related ticket & branch context injection
Reads related Jira tickets and open branches before generating, so the model avoids duplicating existing patterns or conflicting with in-flight work.
Available Now
PR comment interaction (planned)
FlowForge will be able to read reviewer comments on a PR and self-apply requested changes — eliminating the back-and-forth review loop for simple change requests.
Coming Soon
✅

Code Quality & Correctness

Catch defects before human review
Ticket quality gate before generation
Validates that a spec meets readiness criteria before code generation begins. Blocks low-quality tickets early — before developer time is spent on review of broken output.
Available Now
4-verifier + Devil's Advocate compliance check
Architecture, Security, QA, and Go Convention verifiers review every generated PR. A Devil's Advocate agent then stress-tests the consensus — catching approvals that should be challenged.
Available Now
Java automation tests generated alongside Go code
Selenium/JUnit test cases for new endpoints are generated in the same pass as the Go implementation — no manual handoff to the QA team for standard cases.
Available Now
Eval suite for quality regression detection (planned)
A structured test harness to run against FlowForge itself — catches agent/skill regressions when prompts, models, or configs change before they affect production usage.
Coming Soon
📊

Observability & CI Automation

Keep teams informed without manual monitoring
Claude usage cost tracking → Prometheus
Every FlowForge session pushes token cost to Prometheus. Teams can track AI spend per ticket, per engineer, and per feature — and optimize lane choices against actual cost data.
Available Now
Automated code review on every PR
Claude Code reviews all non-dependabot PRs automatically. Provides immediate feedback on style, correctness, and CIL patterns — before human reviewers engage.
Available Now
GitHub Actions post-back to Jira on CI failure
When a CI workflow fails on a FlowForge branch, Jira is updated automatically with failure context. No need to manually ping engineers or update ticket status.
Available Now
Session telemetry for /ff.require pipeline (planned)
Structured telemetry capturing spec quality scores, time-in-stage, and agent decisions during requirement capture — enables data-driven improvement of the spec process.
Coming Soon
