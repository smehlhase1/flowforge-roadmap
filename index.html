Strategic Direction

FlowForge transforms how simplesurance engineers build CIL integrations. The platform replaces weeks of hand-written boilerplate with an AI-guided workflow: from business requirements and discovery, through spec generation and code scaffolding, to automated code review and partner onboarding. Every phase eliminates a friction point — reducing lead time per integration from weeks to hours, making quality and compliance non-negotiable defaults rather than manual checklist items.

How This Changes the Developer Workflow
1
Write a spec once
A single Markdown spec in specs/ defines the feature. FlowForge reads it to understand tasks, contracts, and acceptance criteria.
2
Run /flowforge.start
Generates task breakdown tickets in Jira, creates the feature branch, and caches shared context once — so per-task agents don't re-read the repo.
3
Iterate with /flowforge per task
Each CIL task generates its code artefacts: handler, service, repository, tests, proto changes, and ACL stanzas. Constitution-governed — no off-spec outputs.
4
Open PR with /flowforge.pr
Creates a fully formatted GitHub PR with description, test commands, coverage notes, and linked Jira tickets — no copy-pasting required.
5
CI tests + Windows parity (coming)
Proper GitHub Actions CI tests and full Windows-first parity so engineers on all platforms get identical FlowForge behaviour.

Strategic Rationale
Eliminating the engineering bottleneck in partner onboarding
Today, onboarding a new insurance partner requires a senior Go engineer to: understand the partner's LoB and endpoint topology, manually write catalog stanzas, configure ACL and rate-limiter rules, scaffold an adapter skeleton, write auth client manifests, and create Jira tickets for the implementation work. This is 2–3 days of boilerplate that any skilled engineer resents and any non-engineer cannot do at all. /flowforge.integration replaces that entire sequence with a single guided workflow — moving the bottleneck from engineering capacity to partner readiness.
Today: Partner Onboarding
Senior engineer required for all artifact generation
Manual catalog stanza, ACL, rate-limiter config
Auth client manifest written from scratch
Adapter skeleton copy-pasted from previous partner
Jira tickets created manually — often incomplete
No self-service status visibility for non-engineers
Typical lead time: 2–3 engineering days per partner
With /flowforge.integration
Integration manager runs guided CLI — no Go expertise needed
Partner identity, LoB, endpoint topology collected via prompts
All artifacts generated: catalog, ACL, rate-limiter in minutes
Auth client manifest & adapter skeleton auto-generated
Jira tickets with full AC definition created automatically
